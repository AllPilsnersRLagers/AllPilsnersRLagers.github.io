Install NGINX¶

In order to have a web server in production mode the approach would be different (find more information on the official website of NGINX)but just for testing purposes:

$ sudo apt install nginx

It will ask for confirmation, then start installing the package. After that we can confirm the installation with:

$ sudo nginx -v

You will get the version of the NGINX installed. Then we can start the service with:

$ sudo systemctl start nginx.service

And check the service is running with:

$ systemctl status nginx

You are done if you see “active (running)”! Test with cURL or a browser with the server public IP:

$ curl http://<public_ip>

6/10/26¶

  • Finally got my OCI server to display a nginx webserver example page

  • Issue was i needed to open port 80 using iptables, not just VSN on OCI dashboard

To allow port 80 to access my site I had to define a new rule in the iptables rules file on the server. Navigate to /etc/iptables/rules.v4 and add a new line;

-A INPUT -p tcp -m state --state NEW -m tcp --dport 80 -j ACCEPT

6/12/26¶

  • Used uv run zensical build on the directory of .md files for my blog

  • Moved the generated files in the automatically created site directory to the server using croc send on the client, and sudo croc on the server

  • I moved the whole site directory over to the server in ~/var/www/html/site

  • I added the /site to the default root location in nginx by moding a server rule found in /etc/nginx/sites-enabled/default

server { listen 80 default_server; listen [[::]]:80 default_server;

root /var/www/html/site;

index index.html index.htm index.nginx-debian.html;


server_name _;
  • This will make it easier to rebuild the website after deleting the old zensical build site dir (from client and server). Then running uv zensical build and moving that newly created directory to the server.
